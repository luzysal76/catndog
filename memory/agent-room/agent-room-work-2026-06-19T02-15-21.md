### 구현 계획 (수정 파일 및 변경 내용)

#### 1. `vite.config.js` — manualChunks 추가
```js
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        'vendor-react': ['react', 'react-dom', 'react-router-dom'],
        'vendor-charts': ['recharts'],
        'vendor-pdf': ['jspdf', 'html2canvas'],
      }
    }
  }
}
```

#### 2. `src/App.jsx` — React.lazy + Suspense로 6개 페이지 동적 import
- `import Home from ...` → `const Home = lazy(() => import('./pages/Home.jsx'))`
- `<Routes>` 를 `<Suspense fallback={<Loading/>}>` 로 감싸기

#### 3. `src/lib/pdfExport.js` — dynamic import
- static `import jsPDF` / `import html2canvas` 제거
- `exportToPDF` 함수 내부에서 `const { default: jsPDF } = await import('jspdf')` 방식으로 변경

#### 4. `src/pages/HealthReport.jsx` — recharts dynamic import
- 정적 recharts import 제거
- `const [ChartComponents, setChartComponents] = useState(null)` + useEffect에서 동적 로드
- 또는 recharts 컴포넌트만 별도 파일 `src/components/WeeklyCharts.jsx`로 분리 → lazy import

#### 5. `src/pages/Home.jsx` — 이중 쿼리 수정
- `TodaySummaryCard` props를 `petId` → `records`로 변경
- `Home` 컴포넌트에서 1번만 쿼리 후 `<TodaySummaryCard records={todayRecords}>`로 전달

#### 6. `src/pages/Record.jsx` — capture 제거
- `capture="environment"` 삭제 → 카메라/갤러리 모두 선택 가능

#### 7. `src/components/RecordItem.jsx` — 공통 컴포넌트 추출 (신규)
- Home.jsx `RecentRecordItem` 로직 이동
- Calendar.jsx의 record 렌더링에서 재사용

#### 8. `src/pages/Settings.jsx` — db.js 직접 import 제거
- `addPet/updatePet/deletePet` import 제거
- `usePetContext`에서 `reloadPets` 대신 직접 db 함수 호출 부분을 정리 (현재 구조 유지하되 중복 reload 제거)

### 🟡 발견된 미세 이슈 (블로커 아님)

**HealthReport.jsx recharts 처리 방식:**
- `HealthReport.jsx` L1-5는 recharts를 여전히 **정적 import** 함
- 그러나 `App.jsx`의 `React.lazy`로 HealthReport 페이지 자체가 lazy chunk가 되고, `vite.config.js`의 `manualChunks: 'vendor-charts'`로 recharts가 별도 chunk에 분리됨
- **결과**: `/report` 방문 시에만 두 chunk(page + vendor-charts)가 로드 → 목표한 효과와 동일
- 단, 코드만 보면 "왜 동작하는지" 직관적이지 않음 (주석으로 보완하면 좋음)

- 번들에 포함되지 않으므로(tree-shaking) 실제 문제는 없음
